Получение набора случаев из LIS по идентификаторам

Получение набора случаев из LIS для их обновления на стороне PACS, либо для их выгрузки в PACS по его инициативе.

При поиске всегда передаётся массив, где каждый элемент один или несколько идентификаторов для фильтрации.

При передаче нескольких условий поиска в рамках одного объекта, сервер должен искать объект, который удовлетворяет всем условиям поиска (AND):

[
  // Ищем случай, где id_случая=B001 И номером_заказа=345
  { "caseId": "B001", "orderNumber": "345" },

  // Также ищем случай, где id_случая=B002
  { "caseId": "B002" }
]
Authorizations:
BasicAuth
Request Body schema: application/json
Array
caseNumber
string

Уникальный номер случая

orderNumber
string

Уникальный номер заказа

bioMaterialId
string

Уникальный идентификатор биоматериала в рамках случая. Например, ID пробы (контейнера)

Responses

https://lis.acme.corp/some-prefix/case/search

Импорт списка импортированных ММК в LIS

ЛИС должен принимать от OneCell набор ММК (мединиского медиа контента - микро, макро слайдов и документов) и сохранять в своей базе данных для последующего вывода врачу прямых ссылок на слайды и документы в интерфейсе OneCell.

Authorizations:
BasicAuth
Request Body schema: application/json
Array
caseNumber
required
string

Уникальное название (ID) мед. случая в ЛИС

required
Array of objects (MmcDto)

Полный список ММК данного медицинского случая

Responses

https://lis.acme.corp/some-prefix/case/mmc

    • "mmc": [
      • {
        • "contentId": "string",

        • "url": "http://example.com"
        }
      ]
    }
]

Импорт мед. заключения в LIS

Создание/обновление медицинского заключения в PACS OneCell

Authorizations:
BasicAuth
path Parameters
caseNumber
required
string

Уникальный идентификатор мед. случая

Request Body schema: application/json
macroDescription
string^(?!\s*$).+

Макроописание (будет обновлено в мед. случае)

microDescription
string^(?!\s*$).+

Микроописание

report
string^(?!\s*$).+

Заключение

comment
string^(?!\s*$).+

Доп. замечания и рекомендации

complexity
integer [ 1 .. 5 ]

Категория сложности

object (DictionaryRef)

Ссылка на значение в справочнике

Array of objects (Icd10WithAdditionalCodes)

Поле обязательно, если icd10 не указано

object (DictionaryRef)

Ссылка на значение в справочнике

required
object (UserRef)

Ссылка на пользователя в Active Directory

object (UserRef)

Ссылка на пользователя в Active Directory

ihcResult
string^(?!\s*$).+

Результат ИГХ

additionalResearchResult
string^(?!\s*$).+

Результат доп. исследований

signed
required
boolean

Факт подписания заключения (валидации). Для отмены валидации необходимо отправить весь объект заключения с установленным значением signed=false

signedAt
string <date-time>

Время подписания заключения (валидации). Может использоваться для расчёта оставшегося времени, в течение которого доступна отмена валидации заключения.

locked
boolean

Факт блокирови редактирования заключения. Если true, редактирование заключения в интерфейсе PACS OneCell будет заблокировано. Для отмены блокировки необходимо выгрузить заключение из LIS в PACS со значением locked=false

Responses

https://lis.acme.corp/some-prefix/case/{caseNumber}/medical-report
